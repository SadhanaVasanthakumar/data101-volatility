Why This Question Matters

Every quarter, Meta, Apple, Amazon, Netflix, and Alphabet publish their earnings results. In the hours before and after each release, these stocks move in ways that traders treat as predictable — options prices on these names routinely spike before earnings because the market expects something unusual to happen. But is that expectation backed up by the actual data?

We wanted a simple answer to a simple question: are the five trading days around an earnings announcement actually more volatile than any other week? Not "does volatility feel higher" — does the math say it's higher, consistently, across six years of data?

To answer it properly we used three different statistical methods: a classical hypothesis test that asks whether the difference is real or just noise, a time-series model (GARCH) that accounts for the fact that volatile days cluster together, and a Bayesian extension that gives honest uncertainty ranges instead of false precision. All the computation is in R; the code is included and reproducible.

"The core question: in the five trading days around an earnings call, do these stocks move measurably more than usual — and how long does that elevated volatility last?"

What We Actually Built

This is not just a data analysis — it is a full reproducible pipeline. The project has three layers: an R analysis engine (analysis.R) that handles all the heavy statistical lifting using the quantmod, rugarch, and moments packages; a lightweight Python Flask server (app.py) that serves this website and calls the R script on demand; and this interactive frontend that presents everything with live charts.

The R script downloads six years of daily price data from Yahoo Finance, computes log returns, labels the earnings windows, runs one-tailed Welch's t-tests per ticker, fits GARCH(1,1) models, and evaluates rolling forecast accuracy. A separate Bayesian GARCH was run offline using PyMC for NFLX. Every result is written to a JSON file that powers this dashboard — so the site can be read as both a paper and an interactive tool.

The Data — Six Years of Prices

We downloaded daily adjusted closing prices for META, AAPL, AMZN, NFLX, and GOOGL from January 2019 through December 2024 using R's quantmod::getSymbols() function, which pulls from Yahoo Finance. This gave us approximately 1,508 trading days per ticker — roughly 7,540 data points in total.

From daily prices we computed log returns — basically, the percentage change from one day to the next, expressed in a slightly different math form: r_t = ln(P_t / P_{t−1}). We use log returns rather than simple percentage changes because they handle large moves more symmetrically and behave better in statistical models. If a stock falls 50% and then rises 100%, simple returns add up incorrectly — log returns do not. Each ticker yielded 1,507 return observations over the six-year period.

For each company we manually collected 24 quarterly earnings dates (2019–2024) from Nasdaq's public calendar. We defined an earnings window as the 3 trading days before the announcement, the announcement day itself, and 1 trading day after — 5 days per quarter, around 120 days per ticker across the full period. The pre-announcement days matter because information leaks early and traders position ahead of the number. The single post-announcement day captures the immediate reaction. Everything outside those windows is "normal" trading and forms the comparison group.

The period 2019–2024 was deliberately chosen to include diverse market regimes: the 2020 COVID crash and V-shaped recovery, the 2021 meme-stock bubble and tech euphoria, the 2022 rate-hike-driven tech selloff (NFLX fell 75%, META fell 65%), and the 2023–24 AI-driven recovery. This variety stress-tests whether earnings effects are robust or regime-dependent.

Fat Tails and Why Normal Fails

Before any formal testing, we looked at the distribution of daily returns. The first thing that stands out is that none of these stocks behave like a normal distribution — and that turns out to matter a lot for how we model them.

Every ticker shows negative skewness. In plain terms: big down days happen more often and are more extreme than equivalently-sized up days. The distribution isn't a symmetric bell curve — it's skewed to the left. NFLX has the most extreme skewness at −2.48. The April 2022 session — NFLX fell 35% after reporting its first subscriber loss — is the dominant driver of that figure. META comes in second at −1.20, reflecting its own February 2022 crash. In both cases, the downside event had no equivalent upside counterpart in six years of data. Positive surprises rarely cause equivalent 35% daily gains.

All five also show large excess kurtosis — the tails of the distribution are "fatter" than a normal distribution would predict. NFLX's excess kurtosis is 41.4 — far beyond what a normal distribution would allow. This is driven largely by the April 2022 session where NFLX fell 35% in a single day; one extreme observation can dominate the kurtosis statistic in a fat-tailed series. META comes in second at 24.3. AAPL, AMZN, and GOOGL are milder at 5–6, more consistent with a typical large-cap equity. Extreme moves happen far more often than a normal distribution predicts across all five, but NFLX is in a different category entirely.

This empirical observation — fat tails and volatility clustering — is the core motivation for GARCH modelling. In a GARCH model, large shocks today make large shocks tomorrow more likely, and the conditional variance is tracked explicitly day by day rather than assumed constant.

Hypothesis Testing — What We Tested

With the data in hand, we formalized our question as a one-tailed statistical hypothesis test. The logic is straightforward: we have two groups of observations for each stock — absolute daily returns during earnings windows, and absolute daily returns during all other days. We want to know whether the first group has a significantly higher mean than the second.

We use absolute returns as our volatility proxy rather than squared returns, because they are less sensitive to outliers and easier to interpret in percentage terms. An absolute return of 2.5% means the stock moved 2.5% that day, up or down — that is our measure of "how volatile was today."

We chose Welch's t-test rather than the standard Student's t-test because the two groups have unequal variances (earnings days are inherently more dispersed), and Welch's test does not assume equal variances. It adjusts the degrees of freedom accordingly, giving a more accurate p-value. Significance was set at α = 0.01 — a strict threshold, requiring strong evidence before rejecting the null.

The test is one-tailed: we are only interested in whether earnings weeks have higher volatility, not lower. We have theoretical reasons to expect higher volatility (information release, analyst reactions, algorithmic trading) and no reason to expect lower.

Results — All Five Reject H₀

When we ran the analysis on the actual six-year dataset, the results were stronger than the initial draft predicted. All five tickers reject H₀ at α = 0.01. The degree of the effect varies considerably — and the story behind each ticker explains why.

✓META — earnings vol 3.21% vs normal 1.63% — p = 0.00016, Δ = +1.58 pp. The largest absolute gap in the dataset. Meta's business went through dramatic regime shifts: the Metaverse pivot in 2021, the catastrophic 2022 earnings where the stock fell 26% in a single day, and the "Year of Efficiency" in 2023 that drove one of the fastest large-cap recoveries on record. Each created extreme uncertainty heading into earnings that is simply absent in normal weeks.
✓AAPL — earnings vol 1.75% vs normal 1.33% — p = 0.006, Δ = +0.41 pp. Significant, but the smallest delta. Apple is the most "priced-in" earnings name in markets — supply chain data is practically public, iPhone cycles are widely tracked, analyst consensus is tight. Even so, the data shows a real and consistent bump. The market cannot fully neutralize earnings uncertainty even for the most closely watched stock in the world.
✓AMZN — earnings vol 2.73% vs normal 1.44% — p < 0.00001, t = 5.11, Δ = +1.29 pp. The sharpest t-statistic of any ticker. Amazon is two businesses — retail e-commerce and AWS cloud computing — with different economic drivers and margin profiles. Results regularly shock on one dimension while disappointing on another, producing outsized reactions even when headline revenue lands in-line.
✓NFLX — earnings vol 3.07% vs normal 1.73% — p = 0.003, Δ = +1.34 pp. Netflix is the canonical "one-metric company" — subscriber count drives everything. That single number can swing the stock ±20% depending on a miss or beat by a few hundred thousand users. The data confirms what options traders have priced in for years.
✓GOOGL — earnings vol 2.29% vs normal 1.31% — p = 0.00001, t = 4.57, Δ = +0.98 pp. Our initial draft predicted GOOGL would fail to reach significance (placeholder p = 0.089). The real result inverted that completely. Alphabet's revenue diversification doesn't neutralize earnings risk — it just makes the sources of surprise harder to predict, which is arguably a larger problem for the market.
"Our initial draft expected 3 of 5. Every ticker cleared the bar. The weakest result — AAPL at p = 0.006 — is still six times more significant than the conventional threshold. Earnings move these stocks. That's no longer a question."

GARCH — Volatility Has Memory

The t-test tells us whether earnings weeks are more volatile on average — but it treats every day as independent, with no memory. In reality, turbulent days cluster: a big move today makes a big move tomorrow more likely. Anyone who lived through March 2020 or the 2022 tech selloff experienced this. The GARCH(1,1) model (the name stands for Generalized Autoregressive Conditional Heteroskedasticity — you don't need to memorize that) was built specifically to capture this clustering by tracking a time-varying estimate of risk that updates every day based on recent history.

The model tracks three things to estimate today's risk: ω (a baseline level of volatility that always pulls things back), α (how much yesterday's surprise — a big move — feeds into today's risk estimate), and β (how much of yesterday's risk level carries over directly). Think of it as a weighted average: the market has a long-run "normal" risk level, each day's shock adds some new information, and yesterday's risk estimate persists with weight β. R's rugarch package estimates all three by maximum likelihood.

The GARCH results reveal a more nuanced picture than the initial draft suggested. All five tickers show strong volatility persistence, but the parameters vary meaningfully across names. NFLX has the highest persistence (α + β = 0.993) and the lowest news reactivity (α = 0.019): once NFLX volatility is elevated — say, after an earnings miss — it decays toward baseline very slowly, over many weeks. AMZN has the sharpest news term (α = 0.171): it reacts most strongly to new shocks, but also mean-reverts faster (α + β = 0.949). GOOGL mean-reverts fastest of the group (α + β = 0.923), meaning earnings-driven volatility dissipates relatively quickly for Alphabet compared to the others.

Fig. 3 · α + β persistence by ticker. R's rugarch::ugarchfit(sGARCH(1,1)), α (amber) and β (blue) shown separately. Range: GOOGL at 0.923 (fastest reversion) to NFLX at 0.993 (slowest). AMZN's α = 0.171 stands out as the most news-reactive model.

The rolling out-of-sample RMSE (1,000-day training window, 20-day forecast horizon, 10 windows per ticker) ranged from 1.13 pp for AAPL to 2.21 pp for META, measured in percentage-point units since R scales returns ×100 before fitting. These values reflect the genuine difficulty of forecasting realized volatility — which is measured as |r_t|, a noisy single-day proxy. The GARCH conditional variance is not meant to predict each day's exact move; it tracks the underlying risk regime, which is a harder and more useful thing to do.

The Bayesian Extension

The GARCH fit gives us point estimates for α, β, and ω. But how confident should we be in those numbers? If our dataset were slightly different — a different 6-year window, or different earnings shocks — would the parameters be materially different? Frequentist MLE cannot answer this question. Bayesian inference can.

We specified a Bayesian GARCH model for NFLX — the highest-volatility ticker, chosen because it provides the most information about extreme volatility dynamics. The sampler configuration was 800 posterior draws across 2 NUTS chains with a target acceptance rate of 0.90, run on a 500-day NFLX return window. A note on transparency: the full PyMC NUTS sampler is computationally expensive and was run as a one-time offline step. The posterior results reported here come from that run; they are not re-computed on each page load. The analysis.R script documents this explicitly with a note that the Bayesian block contains cached results. We flag this rather than quietly hard-coding it because reproducibility matters.

The priors were chosen to encode financial domain knowledge: Beta(2,5) for α, which places most probability mass below 0.2 (news terms are typically small); and Beta(5,2) for β, which places most mass above 0.8 (persistence is typically high). These are not "uninformative" priors — they encode what financial econometricians know about GARCH parameters from 40 years of literature.

The posterior means were α = 0.108 [0.092, 0.124] and β = 0.878 [0.851, 0.903] at 95% HPD (Highest Posterior Density) intervals. The intervals are narrow — 500 days of data is sufficient to identify both parameters with precision. The sum α + β = 0.986 sits inside the interval [0.960, 0.998], confirming near-integration even accounting for parameter uncertainty.

The most practically important result: Bayesian prediction intervals captured 93% of out-of-sample realised volatility, compared to 86% for frequentist GARCH. The 7-percentage-point gap arises because the Bayesian model propagates parameter uncertainty forward into its forecasts. When you're not sure whether α is 0.095 or 0.120, your forecast distribution should be wider — and a Bayesian model makes it wider automatically. A frequentist model pretends it knows α exactly, producing overconfident intervals.

"For a risk desk pricing options or setting margin requirements around NFLX earnings: the 7pp coverage gap means the frequentist model understates tail risk in roughly 7% of trading weeks. That's not academic — it's real exposure."

Practical Implications

Taken together, these results have clear practical implications for anyone with exposure to these names around earnings dates.

For options traders: selling short-dated straddles into META, AMZN, or NFLX earnings is selling volatility that is statistically elevated. The t-test tells you the premium is justified. But the GARCH model tells you that post-earnings elevated volatility does not snap back to baseline by Friday — it decays slowly over several weeks. If you sell the straddle and then forget about it, you may be carrying more gamma risk than you think.

For risk managers: standard Value-at-Risk models that assume constant volatility will systematically underestimate risk in earnings windows for META, AMZN, and NFLX. A simple fix: multiply the volatility estimate by the observed ratio (earnings vol / normal vol) for the relevant stock in the 5-day window around its earnings date. For NFLX, that ratio is 3.07 / 1.73 = 1.78 — a 78% upward adjustment to the baseline volatility estimate during the five-day earnings window.

For portfolio construction: Every name in this group shows a real earnings effect, but the magnitude matters for hedging decisions. META and NFLX show the largest absolute gaps — earnings weeks are roughly twice as volatile as normal weeks. AAPL and GOOGL show more modest effects: significant in aggregate, but the delta is small enough (0.41 pp and 0.98 pp respectively) that the cost of hedging may exceed the incremental risk in many practical contexts.

Limitations and What's Next

Earnings timing noise: We use the scheduled announcement date, but many earnings are released after market close, meaning the actual price reaction occurs on the next trading day. This introduces a one-day lag that our 5-day window captures imperfectly. Future work could classify each announcement as pre-market or post-market and adjust the window accordingly.

GARCH symmetry assumption: The sGARCH(1,1) specification treats positive and negative shocks identically. In practice, negative earnings surprises tend to cause larger volatility spikes than equivalent positive surprises — the "leverage effect." Extensions like GJR-GARCH (which adds a term for negative-shock asymmetry) or EGARCH (which models log-variance) would capture this more accurately.

Bayesian computational cost: Our Bayesian model runs on a 500-day NFLX subset. Extending this to all five tickers and the full 1,500-day period would require either variational inference (faster but approximate) or GPU-accelerated HMC sampling. This is tractable with current tools but was beyond the scope of this project.

No economic measures: We measure volatility purely statistically. Future work could connect this to implied volatility (from options prices), which would allow direct comparison between the market's expectation of earnings volatility and the realized volatility we measure here. The gap between implied and realized is the core of earnings options trading strategies.

What Are We Testing?

We want to know whether stock prices move more — are more volatile — in the week around an earnings announcement versus a typical week. We measure volatility using the absolute daily log return, written |r_t|. A large |r_t| means the stock moved a lot that day (in either direction).

H₀ (null): μ(|r_t| earnings weeks) = μ(|r_t| normal weeks)
The null hypothesis says earnings weeks are no different — the average absolute return during the 5-day earnings window equals the average absolute return on all other days. We assume this is true unless the data gives us strong evidence otherwise.
H₁ (alternative): μ(|r_t| earnings weeks) > μ(|r_t| normal weeks)
The alternative says earnings weeks are more volatile — we expect higher average absolute returns. This is one-tailed because we only care about the direction: more volatile, not less.
Welch's t-test: t = (x̄₁ − x̄₂) / √(s₁²/n₁ + s₂²/n₂)
We use Welch's t-test (not Student's) because the two groups have unequal variance — earnings days are inherently more spread out. Welch's adjusts the degrees of freedom to compensate. x̄₁ = mean earnings vol, x̄₂ = mean normal vol, s² = variance, n = sample size.
Significance: α = 0.01 → reject H₀ when p-value < 0.01
We require a 1% significance level — stricter than the conventional 5% — because financial data is noisy and we want to be conservative. A p-value of 0.002 means: if H₀ were true, we'd see data this extreme only 0.2% of the time. That is strong evidence to reject H₀. A p-value of 0.089 means: we'd see this 8.9% of the time even if there were no earnings effect — not convincing enough.

03GARCH(1,1) ModelR · rugarch · sGARCH · MLE
Why GARCH? The t-test ignores the fact that volatility is autocorrelated — a big move today predicts a big move tomorrow. GARCH(1,1) explicitly models this "volatility clustering" by tracking a time-varying conditional variance σ²_t that depends on past returns and past variance estimates. It is the standard model for daily financial return volatility.
r_t = μ + ε_t     ε_t = σ_t · z_t     z_t ~ N(0,1)
σ²_t = ω  +  α · ε²_{t−1}  +  β · σ²_{t−1}

ω = long-run variance baseline (intercept)
α = ARCH term: how much does yesterday's squared shock increase today's variance? (news impact)
β = GARCH term: how much does yesterday's variance forecast carry into today? (persistence/memory)
α + β = total persistence — the closer to 1, the slower volatility reverts to baseline

Why go Bayesian? Frequentist MLE gives you one number for each parameter (e.g., α = 0.108). But that number has uncertainty — a different 500-day window would give a slightly different answer. Bayesian inference gives you a probability distribution over each parameter, quantifying exactly how uncertain you should be. This propagates into wider, more honest prediction intervals.

PyMC model — recursive GARCH via pytensor.scan
import pymc as pm, pytensor.tensor as pt

with pm.Model() as model:
  omega = pm.Exponential("omega", lam=1.0)
  alpha = pm.Beta("alpha", alpha=2, beta=5)
  beta  = pm.Beta("beta",  alpha=5, beta=2)
  s0    = pm.Exponential("s0", lam=0.1)

  def garch_step(r, s2, w, a, b):
      return w + a*r**2 + b*s2

  s2_seq, _ = pytensor.scan(
      fn=garch_step,
      sequences=[pt.as_tensor(r_bay[:-1])],
      outputs_info=[s0],
      non_sequences=[omega, alpha, beta])

  pm.Normal("obs", mu=0,
      sigma=pm.math.sqrt(s2_seq),
      observed=r_bay[1:])
  trace = pm.sample(800, tune=400,
               target_accept=0.9)

06How to Run This ProjectSetup · Files · Local machine · GitHub Pages

What Each File Does

analysis.R
R · ~350 lines
The entire statistical analysis. Downloads six years of daily prices from Yahoo Finance, computes log returns, labels earnings windows, runs Welch's t-tests, fits GARCH(1,1) models, evaluates rolling forecast accuracy, and writes everything to outputs/results.json. This is the only file that does any real computation. Run it once and leave it — results are cached.
index.html
HTML · self-contained
The entire website — this page. All charts, tables, the blog, and the R console live in this one file. It has the real analysis results baked in as fallback data, so it works even with no server running. When it can find outputs/results.json, it loads that instead for the live computed values.
outputs/results.json
JSON · ~40 KB
The pre-computed output from analysis.R. Contains all hypothesis test results, GARCH parameters, rolling volatility series, RMSE values, correlation matrices, and normalized prices. The website reads this file to populate every chart and table. You can re-generate it at any time by re-running analysis.R.
app.py
Python · optional
A lightweight web server — optional. Serves index.html locally and exposes an endpoint that re-runs analysis.R on demand. You only need this if you want to re-run the R analysis from the browser using the ⟳ button. The site works perfectly without it.

Download index.html and outputs/results.json (keep them in the same folder structure). Then just double-click index.html. The site will open in your browser with all charts and data already loaded from the embedded static values.

UpdatingRe-running the analysisWhen to re-run · what changes
The results.json in the repo was generated in April 2026 covering Jan 2019 – Dec 2024. If you want fresher data — for example to include 2025 earnings — edit the END_DATE variable at the top of analysis.R and re-run it. 
# In analysis.R, line ~5 — change the date range:
START_DATE <- "2019-01-01"
END_DATE   <- "2025-12-31"  # extend to include 2025

# Then re-run:
Rscript analysis.R

# A new outputs/results.json is created.
# The website auto-loads it on next page visit.
# To publish: push the new results.json to GitHub.
If you add new tickers, also add their quarterly earnings dates to the earnings_dates list in analysis.R. The analysis uses manually curated dates — it cannot infer them automatically.
